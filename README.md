# VK & Yandex Music to Spotify

[![License: MIT](https://img.shields.io/badge/License-MIT-4776e6.svg)](./LICENSE)

Export tracks from VK Music and Yandex Music to Spotify Liked Songs with a Chrome extension and a Python importer.

Инструмент переносит список любимой музыки из **Яндекс Музыки** или **VK Музыки** в библиотеку Spotify в два шага:

1. Chrome-расширение собирает названия и исполнителей в `tracks.txt`.
2. Python-скрипт ищет эти треки в Spotify и добавляет найденные в «Любимые треки».

Аудиофайлы не скачиваются. Расширение работает только с текстом треков, видимых в авторизованной вкладке.

![How it works](./docs/workflow.svg)

## Структура

```text
extension/          распакованное расширение Chrome
spotify-importer/   Python-скрипт импорта в Spotify
```

## 1. Установить расширение

1. Откройте `chrome://extensions`.
2. Включите «Режим разработчика».
3. Нажмите «Загрузить распакованное расширение».
4. Выберите папку [`extension`](./extension).
5. Откройте список «Мне нравится», плейлист или «Мои треки» в VK Музыке / Яндекс Музыке.
6. Нажмите значок расширения и кнопку «Собрать и скачать tracks.txt».

Расширение прокрутит виртуальный список до конца и предложит сохранить файл. При остановке оно сохранит уже собранную часть.

## 2. Настроить Spotify

1. Откройте [Spotify Developer Dashboard](https://developer.spotify.com/dashboard), создайте приложение и скопируйте Client ID и Client Secret.
2. В настройках приложения добавьте Redirect URI: `http://127.0.0.1:8888/callback`.
3. Перейдите в папку `spotify-importer` и установите зависимости:

   ```powershell
   py -m pip install -r requirements.txt
   ```

4. Скопируйте `.env.example` в `.env` и вставьте свои значения. Файл `.env` не попадёт в Git.
5. Скопируйте скачанный `tracks.txt` в папку `spotify-importer`.

## 3. Импортировать

В папке `spotify-importer` выполните:

```powershell
py import_liked_tracks.py
```

При первом запуске откроется авторизация Spotify. Скрипт сохраняет результат каждого поиска в `.spotify-import-state.json`, поэтому после остановки или ошибки следующий запуск продолжится с того же места, не повторяя уже выполненные поисковые запросы. Найденные треки добавляются через актуальный API Spotify пачками по 40; ненайденные записываются в `not_found.txt`.

По умолчанию порядок строк переворачивается — это сохраняет поведение исходного скрипта. Если порядок нужен как в файле, используйте:

```powershell
py import_liked_tracks.py --keep-order
```

Если нужно сначала только собрать совпадения, а добавить позже, используйте:

```powershell
py import_liked_tracks.py --resolve-only
```

Не удаляйте `.spotify-import-state.json`, пока не закончите перенос. Для полного перезапуска с нуля используйте `--reset-state`.

## Ограничения

- VK Музыка и Яндекс Музыка периодически меняют интерфейс. Если экспорт начал собирать 0 треков или не прокручивает список, создайте Issue с сервисом, версией браузера и числом ожидаемых треков. Не прикладывайте cookies, токены, `.env` или приватные ссылки.
- Spotify может подобрать другой трек с тем же названием или не найти редкую версию. Все ненайденные строки сохраняются в `not_found.txt`.
- Расширение предназначено для личного экспорта метаданных, а не скачивания аудио.
